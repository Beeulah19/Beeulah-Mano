Pilabz Heat Pump Dashboard

A real-time embedded dashboard designed for a campus heat pump installation, making technical performance data understandable at a glance for both engineers and non-technical employees.

01 — Problem

Dense engineering data. 3 seconds to read it.

The dashboard had to show six live metrics — COP, electrical power, heat delivered, cumulative savings, and daily averages — on one fixed screen. There was no interaction, scrolling, or second-level detail.
The challenge was to make dense engineering data readable from a distance and meaningful within a few seconds.

02 — Design decisions

Three choices that defined the screen

Hierarchy by impact, not engineering order

COP and cumulative savings were made visually dominant because they communicated the clearest performance and impact story. Raw engineering values remained visible but secondary.

Rejected

An equal-weight grid where every metric had the same visual importance.

Designed for distance

The screen was designed for people standing 1–2 metres away, not sitting at a desk. Large numbers, clear spacing, and grouped cards helped users scan the dashboard quickly.

Rejected

A dense dashboard layout with smaller labels and more detailed values.

Familiar visual language

I used a clean card-based interface with soft translucent surfaces to make the technical data feel approachable for a mixed audience.

Rejected

Opaque industrial metric tiles that felt functional but less inviting for non-technical viewers.

03 — Impact

Numbers that tell a real story

The heat pump replaced a fuel-based heating process with a more efficient system and it was installed as a live always-on display at Zoho, Tenkasi campus. It helped make heat pump performance and energy savings visible in real time.

04 — Reflection

What this project taught me

This project taught me that UX is not always about interaction. In an ambient display, the main job is clarity. The screen has to communicate without asking the viewer to stop, tap, or explore.
